# **「跃动校园 / Campus Go」演示版 (Demo) 产品需求文档**

**文档版本**：V2.1 (Apple Minimalist 极简风 Demo 版)

**目标读者**：前端开发工程师、UI/UX 设计师、产品演示相关人员

**技术栈限制**：React (Vite) \+ Tailwind CSS \+ Framer Motion \+ 纯前端 Mock 数据

## **一、 产品概述与演示目标**

### **1.1 产品定义**

「跃动校园」是一款旨在通过 LBS 盲盒、虚拟宠物陪伴与代币经济，激励大学生打破“不出门”阻力，并促进校园互助与破冰社交的 Web3/游戏化应用。

### **1.2 演示版 (Demo) 核心目标**

本 Demo 版本**不包含**任何真实的后端逻辑、数据库和地理位置反算。

核心目标是产出一个**高保真、极具沉浸感的交互式原型**（部署于阿里云 OSS，套用手机外壳 Mockup），向投资人和种子用户清晰传达：

1. **破冰体验**：如何用雷达和盲盒吸引用户第一次出门。  
2. **社交匹配**：如何通过具象化的高级 UI 降低社交防御。  
3. **劳动力流转**：悬赏大厅如何运转。

## **二、 全局 UI/UX 设计规范 (Apple Minimalist)**

开发在实现时，必须严格遵守以下“苹果原生简约风”的视觉法则：

* **克制的色彩系统**：  
  * **背景色**：极致的纯白 (\#FFFFFF) 或极浅的系统灰 (\#F2F2F7)。  
  * **文字色**：主要依赖纯黑 (\#000000) 与次级灰度 (\#8E8E93)，通过不同的字重（Font Weight: 400/600/800）来拉开信息层级。  
  * **强调色**：全局仅使用一种高亮色（如 Apple Fitness 的亮绿色 \#34C759 或亮蓝色 \#007AFF），仅用于核心操作（开启盲盒、确认接单）。  
* **材质与空间 (Glassmorphism & Spacing)**：  
  * **毛玻璃**：弹窗、悬浮卡片、底部 Tab 栏大量使用毛玻璃效果（Tailwind: backdrop-blur-md bg-white/70）。  
  * **大留白**：模块间避免使用实体分割线，用充分的留白（负空间）进行区隔。  
* **动效反馈 (Framer Motion)**：  
  * 拒绝线性过渡。所有页面切换、弹窗升起、按钮点击，必须使用系统级丝滑的弹簧阻尼效果（Spring Physics），模拟真实的物理惯性。

## **三、 核心功能模块详细说明**

### **3.1 Tab 1: 【首页 / 探索】 —— 破冰与陪伴**

**场景目标**：展示核心的“Aha Moment”，演示用户如何被雷达吸引并获得反馈。

* **\[功能 1\] 沉浸式 LBS 雷达**  
  * **呈现**：背景为高清高对比度的校园地图底图（静态切图）。  
  * **动效**：进入页面触发细线光圈向外扩散扫描。扫描过处，安静浮现清晰的“高亮圆点”（代表热力人群），部分区域离散，核心区聚集。  
* **\[功能 2\] 盲盒触发与退场**  
  * **呈现**：扫描完毕后，地图特定坐标平滑弹出一个极简图标的“新手盲盒”。  
  * **交互**：3 秒后，地图底图触发缩小动效（类似 iOS 关闭 App 退回桌面的丝滑缩放），收缩并停靠至左上角，让出屏幕主视觉。  
* **\[功能 3\] 数字宠物 (Lottie 动画)**  
  * **呈现**：在纯白背景中央，加载预设的 Lottie 动画（代替原有的手账风糯米团）。呈现类似于 Apple Watch 动态表盘的高级质感。  
  * **交互**：点击不同区域支持切换 Lottie 动画状态（如开心、待机）。  
* **\[功能 4\] 极简数据看板**  
  * **呈现**：页面下方展示今日运动数据。采用 Apple Health 风格大号纯黑数字（如 **8,420**），搭配极简的粗线条环形进度条。

### **3.2 Tab 2: 【游乐场 / 派对】 —— 游戏化社交**

**场景目标**：展示丰富的校园活动与真实的组队匹配体验。

* **\[功能 1\] 周末限时派对 (大型卡片)**  
  * **呈现**：横向轮播图。采用 App Store “Today” 推荐页风格的超大圆角、边缘锐利的图片卡片。  
  * **内容**：高清暗色底图上悬浮纯白粗体字，展示：“🐱 猫抓老鼠”、“🙈 校园躲猫猫”、“🧟 末日逃亡”。  
  * **交互**：点击卡片可无缝放大至全屏，展示活动详情介绍。  
* **\[功能 2\] 运动羁绊匹配 (重点演示)**  
  * **交互**：点击“寻找搭档”按钮，屏幕中央出现系统原生 Spinner (菊花加载转轮) 模拟匹配中。  
  * **呈现**：2 秒后，从底部丝滑升起一张**毛玻璃面板**。  
  * **Mock 档案**：使用一张极高清晰度、带极细灰边的真实“学姐”照片作为头像。排版类似 iOS 联系人卡片（大标题姓名，小字号显示“大三 / 外语系”）。下方以极简列表项展示共同目标（如：本周共跑 10km）。

### **3.3 Tab 3: 【集市 / 互助】 —— 劳动力流转**

**场景目标**：展示悬赏大厅的运转与接单流程。

* **\[功能 1\] 悬赏令大厅 (原生列表)**  
  * **呈现**：抛弃手账便利贴。采用类似 iOS 备忘录 (Reminders) 或邮件的纯白极简列表 (List View)，列表项之间采用极浅的 1px 灰线分割。  
  * **Mock 内容**：  
    * 📦 代取快递 (南区菜鸟驿站 → 某女生宿舍)  
    * 🧋 帮买奶茶 (排队代购)  
    * 📚 图书馆占座 (早起互助)  
* **\[功能 2\] 接单交互流**  
  * **交互**：点击某一行列表，平滑向下展开详情区域（显示模糊距离与奖励）。  
  * 点击“接单”高亮按钮后，从底部弹出 iOS 原生风格的 **Action Sheet (确认操作表单)** 询问是否确认接单。确认后，页面顶层滑出系统级极简 Toast 提示（“接单成功，请在 30 分钟内完成”）。  
* **\[功能 3\] 运动搭子局**  
  * **呈现**：精简版列表。干净头像 \+ 粗体标题（如“晚上8点操场夜跑”）+ 极简 Tag。右上角放置一个极简的 \+ 号。

### **3.4 Tab 4: 【大本营 / 我的】 —— 资产与成就**

**场景目标**：展示极简风下的资产管理与荣誉陈列。

* **\[功能 1\] 高级数据仪表盘**  
  * **呈现**：左上角使用巨大字重显示“连续运动天数”，右上角用原生金额样式（类似 $ 1,200）展示“钱包余额/社交代币”。  
* **\[功能 2\] 3D 荣誉陈列架**  
  * **呈现**：不再使用木质书架，改为纯白色背景 \+ **磨砂亚克力玻璃材质展台**。  
  * **内容**：放置预渲染的高清 3D 极简奖杯（如早起鸟勋章），底部带有拟真的 AO 阴影（环境光遮蔽），突出反差高级感。  
* **\[功能 3\] 代币兑换商城**  
  * **呈现**：实体奖励展示参考 **Apple Wallet (钱包凭证)** 的设计。  
  * **内容**：如“瑞幸咖啡 5 元券”，做成极简的 Passbook 卡片样式。点击兑换按钮时，触发 Haptic 模拟震动反馈与 Toast 提示。

## **四、 数据 Mock 与开发约定**

1. **mockData.json 统一管理**：  
   前端建立全局唯一的 JSON 配置文件，将用户的头像、步数、钱包余额、悬赏令列表、匹配到的“学姐”信息统一配置在此处，方便演示前随时修改文案。  
2. **零网络请求限制**：  
   演示版彻底切断真实 API 调用。所有的“加载中”、“签到”、“接单”状态，全部使用 setTimeout 延迟 1-2 秒配合 Spinner 实现模拟。  
3. **移动端适配与外壳**：  
   整体项目需要部署为一个静态网站。在桌面端浏览器打开时，最外层需要强制包裹一个 iPhone 14/15 的 Mockup 边框，超出边框的内容需隐藏 (overflow-hidden)，确保随时随地都能打开演示。
